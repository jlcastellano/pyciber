Fundamentos de Python para Ciberseguridad

BLOQUE 1: Introducción y Entorno

1.1 ¿Por qué Python en Ciberseguridad?

Python se ha convertido en el lenguaje de referencia en ciberseguridad por varias razones fundamentales. Su sintaxis clara y legible permite escribir scripts rápidamente, algo crucial cuando necesitas automatizar tareas repetitivas o crear herramientas personalizadas durante un pentest. La enorme cantidad de bibliotecas disponibles, desde Scapy para manipulación de paquetes hasta Requests para interacción HTTP, proporciona un arsenal completo sin necesidad de reinventar la rueda.

La mayoría de herramientas profesionales están escritas en Python o permiten integración con él. Hablamos de frameworks como Metasploit (que acepta scripts Python), herramientas como SQLMap, Volatility para análisis forense, o incluso gran parte de las utilidades incluidas en Kali Linux. Además, muchos exploits publicados en bases de datos como Exploit-DB están escritos en Python, por lo que entender el lenguaje te permite analizar, modificar y adaptar código existente.

Casos de uso concretos:

  • Automatización de escaneos y reconocimiento
  • Desarrollo de exploits y payloads personalizados
  • Análisis de logs y detección de anomalías
  • Scripting para CTFs (Capture The Flag)
  • Automatización de tareas de administración segura
  • Desarrollo de herramientas forenses
  • Creación de fuzzers y scrapers

1.2 Instalación y Configuración del Entorno

Instalación de Python

En Windows:
Descarga el instalador desde python.org. Durante la instalación, marca obligatoriamente la casilla "Add Python to PATH". Verifica la instalación abriendo PowerShell o CMD y ejecutando python --version.

En Linux (Debian/Ubuntu/Kali):
Python suele venir preinstalado. Verifica con python3 --version. Si necesitas instalarlo: sudo apt update && sudo apt install python3 python3-pip.

En macOS:
Usa Homebrew: brew install python3.

Configuración del Editor

Recomiendo Visual Studio Code por su versatilidad y extensiones. Instala las siguientes extensiones:

  • Python (Microsoft)
  • Pylance para autocompletado inteligente
  • Opcionalmente indent-rainbow para visualizar mejor la indentación

Alternativamente, PyCharm Community Edition ofrece un entorno más completo pero con mayor consumo de recursos.

Verificación del Entorno

# Verificar versión de Python
python3 --version

# Verificar pip (gestor de paquetes)
pip3 --version

# Crear un directorio de trabajo
mkdir ~/curso_python_security
cd ~/curso_python_security

1.3 El Intérprete Interactivo y Scripts

Python ofrece dos formas de trabajar. El intérprete interactivo (REPL) es perfecto para probar código rápidamente y experimentar. Accedes escribiendo python3 en la terminal.

>>> print("Hola, futuro pentester")
Hola, futuro pentester
>>> 2 + 2
4
>>> exit()

Para programas más complejos, escribimos archivos .py. Crea un archivo llamado primer_script.py:

#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
Primer script del curso de Python para Ciberseguridad
Autor: [Tu nombre]
"""

print("=== Iniciando reconocimiento ===")
print("Objetivo: Aprender Python")
print("Estado: Conexión establecida")

Ejecútalo desde la terminal:

python3 primer_script.py
Nota: El shebang (#!/usr/bin/env python3) permite ejecutar el script directamente en sistemas Unix si le das permisos de ejecución (chmod +x primer_script.py).

1.4 — Entorno profesional mínimo: venv + pip + ejecución reproducible

1) ¿Por qué esto importa en ciber?

  • Evita "en mi PC funciona": cada herramienta y librería queda aislada por proyecto.
  • Reduce conflictos de versiones (muy típico con requests, cryptography, etc.).

2) Flujo estándar (Linux/macOS)

mkdir curso-python-ciber
cd curso-python-ciber

python3 -m venv .venv
source .venv/bin/activate

python -V
pip -V

3) Flujo estándar (Windows PowerShell)

mkdir curso-python-ciber
cd curso-python-ciber

py -m venv .venv
.\.venv\Scripts\Activate.ps1

python -V
pip -V

Si PowerShell bloquea scripts: explicar brevemente Set-ExecutionPolicy -Scope CurrentUser RemoteSigned (solo si aplica en el aula).

4) Instalar dependencias y "congelar" versiones

pip install requests
pip freeze > requirements.txt

Reinstalación en otra máquina:

pip install -r requirements.txt

5) Estructura mínima de proyecto (para el curso)

curso-python-ciber/
  .venv/
  src/
    hello_hacker.py
  data/
    sample.txt
  requirements.txt
  README.md

6) Reglas de oro para el aula

  • Siempre activar .venv antes de ejecutar.
  • Ejecutar con el mismo Python del venv: python src/archivo.py
  • No mezclar instaladores del sistema con pip global.

1.5 Variables y Tipos de Datos Básicos

Las variables en Python son contenedores para almacenar datos. No necesitas declarar el tipo explícitamente; Python lo infiere automáticamente.

#!/usr/bin/env python3
"""
Demostración de variables y tipos de datos
"""

# === STRINGS (cadenas de texto) ===
objetivo = "192.168.1.100"
protocolo = 'TCP'
banner = """SSH-2.0-OpenSSH_8.2p1 Ubuntu-4ubuntu0.5
Welcome to Ubuntu 20.04.3 LTS"""

# === INTEGERS (números enteros) ===
puerto = 22
intentos_maximos = 3
timeout = 5

# === FLOATS (números decimales) ===
version_ssl = 3.0
tiempo_respuesta = 0.045

# === BOOLEANS (verdadero/falso) ===
puerto_abierto = True
requiere_autenticacion = False

# === NONE (ausencia de valor) ===
respuesta_servidor = None

# Verificar tipos con type()
print(type(objetivo))       # 
print(type(puerto))         # 
print(type(tiempo_respuesta))  # 
print(type(puerto_abierto)) #

Convenciones de Nomenclatura

En Python seguimos PEP 8 para nombrar variables. Usamos snake_case (palabras en minúscula separadas por guiones bajos). Los nombres deben ser descriptivos: ip_objetivo es mejor que ip o x. Las constantes se escriben en MAYÚSCULAS: MAX_INTENTOS = 5.

# ✓ Correcto
direccion_ip = "10.0.0.1"
numero_puerto = 443
esta_activo = True

# ✗ Evitar
x = "10.0.0.1"
DireccionIP = "10.0.0.1"
direccionip = "10.0.0.1"

1.6 Operadores

Operadores Aritméticos

#!/usr/bin/env python3
"""
Operadores en Python
"""

# Operadores aritméticos
a = 10
b = 3

print(f"Suma: {a + b}")           # 13
print(f"Resta: {a - b}")          # 7
print(f"Multiplicación: {a * b}") # 30
print(f"División: {a / b}")       # 3.333...
print(f"División entera: {a // b}")  # 3
print(f"Módulo (resto): {a % b}") # 1
print(f"Potencia: {a ** b}")      # 1000

# Aplicación: Calcular rango de IPs
ip_base = 192 * (256**3) + 168 * (256**2) + 1 * 256 + 0
print(f"IP base en decimal: {ip_base}")

Operadores de Comparación

puerto = 22
puerto_ssh = 22
puerto_http = 80

print(puerto == puerto_ssh)   # True (igualdad)
print(puerto != puerto_http)  # True (diferente)
print(puerto < puerto_http)   # True (menor que)
print(puerto > 1024)          # False (mayor que)
print(puerto <= 22)           # True (menor o igual)
print(puerto >= 1)            # True (mayor o igual)

Operadores Lógicos

puerto = 443
protocolo = "HTTPS"

# AND: ambas condiciones deben ser verdaderas
es_web_seguro = puerto == 443 and protocolo == "HTTPS"
print(f"¿Es conexión web segura? {es_web_seguro}")  # True

# OR: al menos una condición debe ser verdadera
es_puerto_web = puerto == 80 or puerto == 443
print(f"¿Es puerto web? {es_puerto_web}")  # True

# NOT: invierte el valor
servicio_activo = True
print(f"¿Servicio inactivo? {not servicio_activo}")  # False

Operadores de Asignación Compuesta

contador = 0
contador += 1   # Equivale a: contador = contador + 1
contador -= 1   # Resta y asigna
contador *= 2   # Multiplica y asigna
contador /= 2   # Divide y asigna
